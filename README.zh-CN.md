# Vitals

Vitals 是一个纯客户端 Fabric Mod，为活体实体渲染可配置的 RPG 风格头顶血条。它支持单人游戏，也能直接进入没有安装 Vitals 的多人服务器。

[English](README.md)

## 功能

- 在附近活体实体头顶显示世界空间血条。
- 平滑生命变化和延迟伤害拖尾。
- 精确显示当前/最大生命，可配置小数精度。
- 可选显示实体名称和护甲数值。
- 分别控制玩家、原版 Boss、已驯服、中立、敌对、友好生物、盔甲架和其他活体实体。
- 使用深度测试，血条会被不透明世界方块遮挡。
- 内置中英双语配置面板和实时预览。
- 不使用 Mixin、自定义网络协议或第三方配置库。

## 打开配置面板

- Windows/Linux：**左 Alt + V**
- macOS：**左 Option + V**

只有左侧修饰键会触发快捷键。面板包含“显示”和“实体”两个分页，以及实时预览、恢复默认、取消、应用和完成操作。存在未应用修改时按 Esc，会先询问是否放弃修改。

设置保存在 `config/vitals.json`。保存流程包含临时文件、复读校验、旧配置备份，以及文件系统支持时的原子替换。

## 多人游戏行为

服务器无需安装 Vitals。Mod 读取 Minecraft 已经同步给客户端的生命和最大生命状态。

每位希望看到血条的玩家都需要在自己的客户端安装 Vitals。没有安装 Mod 的玩家不会被推送 UI，客户端之间也不会共享本地配置。

## 支持环境

| 组件 | 版本 |
| --- | --- |
| Minecraft | `1.21.3` |
| Java | `21` |
| Fabric Loader | `0.18.4` 或更新的兼容版本 |
| Fabric API | `0.114.1+1.21.3` 或更新的兼容版本 |

Vitals 是纯客户端 Mod。请将 JAR 和 Fabric API 一起放入客户端的 `mods` 目录。

## 兼容与限制

- 未知 Mod 添加的 `LivingEntity` 默认使用“其他活体实体”分类。
- Twilight Forest（暮色森林）等 Mod 的专用 adapter 会在定向兼容测试后加入。
- 初始版本不显示吸收生命和完整状态效果，因为纯客户端无法可靠取得所有远端实体的这些数据。
- 默认隐藏本地玩家，避免第一人称镜头重叠。
- 尊重原版名称标签；Minecraft 已经渲染名称时，Vitals 不重复绘制名称。
- 若服务器主动隐藏或改写客户端可见生命，仍以服务器行为为准。

## 性能边界

- 可配置距离：`8–64` 格，默认 `32` 格。
- 每帧最多收集 `256` 个附近候选实体，最多渲染 `96` 条血条。
- 动画缓存最多保存 `128` 个实体，并在世界和连接生命周期变化时清空。
- 不强制加载区块，也不修改实体状态。

## 构建

~~~bash
./gradlew clean build --no-daemon --stacktrace
~~~

构建会检查 Java 编译、无依赖逻辑测试、双语 key 一致性、客户端专用 metadata、资源和重映射 JAR。
